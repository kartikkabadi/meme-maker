# Template authoring guide

How curated templates are organized, described, and added. The manifest
(`assets/templates/manifest.json`) is generated — never edit it by hand.

## Layout

```
assets/templates/
  images/               # root image templates (.png/.jpg) + sidecars
  gifs/                 # root GIF templates + sidecars
  packs/<pack>/         # contributor packs (see below)
    pack.json           # optional pack metadata
    images/  gifs/      # media + sidecars
  manifest.json         # generated by `npm run build:manifest`
  thumbs/               # generated by `npm run build:thumbs`
  CREDITS.md            # source + license for every template
```

## The `<id>.meta.json` sidecar

Every media file needs a sidecar with the same basename (e.g. `drake.jpg` +
`drake.meta.json`). Fields (validated against `TemplateSchema` in `src/spec.ts`;
`id`, `type`, `file`, `width`, `height` are derived by the manifest builder):

```json
{
  "name": "Drake Hotline Bling",
  "tags": ["choice", "preference", "two-panel"],
  "category": "choice",
  "slots": [
    {
      "name": "no",
      "rect": [620, 30, 560, 540],
      "hint": "the rejected option",
      "anchor": "middle",
      "align": "center",
      "style": { "color": "black", "strokeWidth": 0 }
    }
  ],
  "source": {
    "url": "https://imgflip.com/memetemplate/Drake-Hotline-Bling",
    "license": "fair-use",
    "attribution": "via Imgflip"
  }
}
```

- `name` (required): human-readable template name.
- `tags` (required): lowercase keywords used by `meme templates search`.
- `category` (optional): coarse grouping shown in listings.
- `slots` (required): named text areas.
  - `name`: slot id referenced by `--text <name>=...` and spec `texts[].slot`.
  - `rect`: `[x, y, width, height]` in source-image pixels.
  - `hint`: tells agents what kind of text belongs in the slot.
  - `anchor` / `align` / `style` (optional): per-slot defaults, overridable per render.
- `source` (optional but strongly encouraged): provenance and license.

## Packs and `pack.json`

Packs let many contributors add templates in parallel without id conflicts.
Each pack is a directory under `assets/templates/packs/` with optional metadata:

```json
{
  "id": "my-pack",
  "name": "My Pack",
  "tags": ["shared", "tag"]
}
```

- `id`: prefix for generated template ids (defaults to the directory name).
- `name`: human-readable pack name.
- `tags`: merged into every template's tags.

For `packs/my-pack/images/foo.png` the manifest builder generates
`id: "my-pack-foo"`, `file: "packs/my-pack/images/foo.png"`, `pack: "my-pack"`.
Sidecars keep the bare filename (`foo.meta.json`).

## Adding a new template pack

1. Create `assets/templates/packs/<pack>/` with `images/` and/or `gifs/`,
   plus a `pack.json` if you want an explicit id, name, or shared tags.
2. Drop media files in and write an `<name>.meta.json` sidecar for each.
3. `npm run build:manifest` — scans `assets/templates/**`, derives ids, paths,
   and dimensions, validates every sidecar, and regenerates `manifest.json`.
4. `npm run build:thumbs` — regenerates `assets/templates/thumbs/` and
   `docs/contact-sheet.webp`.
5. Record each template's source and license in `assets/templates/CREDITS.md`.
6. Verify: `meme templates show <pack>-<name> --json`, then render a test meme
   (see [examples/](../examples/)).
